# M0 迁移指南:今晚跑通 500GB 无人值守迁移

> 原则:全程零弹窗、零确认。中断随意,重跑即续传。

---

## 一、原理速览

- **传输引擎**:rclone(发送端 `rclone copy` + 接收端 `rclone serve sftp`)
- **为什么不会像 Explorer 那样弹窗**:rclone 直接调用文件系统 API,Windows 上自动加 `\\?\` 前缀,绕过保留名(nul/con/aux...)和 260 字符路径限制;`rclone copy` 本身是非交互命令,遇错只记日志、跳过、继续
- **断点续传**:按"大小+修改时间"跳过已完成文件;大文件用 `--partial-suffix` 保留半成品断点
- **无人值守**:`migrate.ps1` 脚本自动循环重跑,直到全部成功(exit code 0)才停

## 二、接收端设置(新电脑,5 分钟)

### 如果新电脑是 Mac

```bash
# 1. 安装 rclone
brew install rclone

# 2. 创建接收目录
mkdir -p ~/Migration

# 3. 启动接收服务(密码请改成自己的,要与发送端脚本一致)
rclone serve sftp ~/Migration --addr :2022 --user lanmigrate --pass ChangeMe2026
```

首次运行 macOS 可能弹一次"允许接受传入网络连接"→ 点允许(仅此一次,之后全程无人值守)。

查看本机 IP:`ipconfig getifaddr en0`

### 如果新电脑是 Windows 11

```powershell
# 1. 安装 rclone
winget install Rclone.Rclone
# (装完重开一个终端让 PATH 生效)

# 2. 创建接收目录
mkdir D:\Migration

# 3. 放行防火墙端口(管理员 PowerShell,一次性)
New-NetFirewallRule -DisplayName "LanMigrate SFTP" -Direction Inbound -LocalPort 2022 -Protocol TCP -Action Allow

# 4. 启动接收服务
rclone serve sftp D:\Migration --addr :2022 --user lanmigrate --pass ChangeMe2026
```

查看本机 IP:`ipconfig`(看无线局域网适配器的 IPv4 地址)

> 接收端窗口保持开着即可。关了也没关系——重新运行第 3/4 步命令,发送端会自动续传。

## 三、发送端设置(旧电脑 Windows 10)

```powershell
# 1. 安装 rclone
winget install Rclone.Rclone
# 若 Win10 没有 winget:去 https://rclone.org/downloads/ 下载 zip,
# 解压后把 rclone.exe 所在目录加入 PATH,或直接放到脚本同目录

# 2. 把 migrate.ps1 和 filters.txt 放到同一个文件夹(如 D:\lanmigrate\)

# 3. 编辑 migrate.ps1 顶部的"配置区":
#    - $SourceDir  = 要迁移的目录(如 D:\)
#    - $RemoteHost = 新电脑的 IP
#    - $RemotePass = 与接收端一致的密码

# 4. 运行
powershell -ExecutionPolicy Bypass -File D:\lanmigrate\migrate.ps1
```

之后就可以走人了。任何时候:
- **Ctrl+C / 合盖 / 断网 / 关机** → 都安全
- **下次继续** → 接收端跑起 serve 命令,发送端重跑 migrate.ps1,自动从断点继续
- **换了局域网/IP 变了** → 只需改脚本里的 `$RemoteHost`,其余照旧

## 四、无人值守的关键参数解释

| 参数 | 作用 |
|------|------|
| `--retries 5 --retries-sleep 15s` | 整轮失败自动重试 5 次(应对短暂断网) |
| `--low-level-retries 20` | 单个网络请求级重试(应对 Wi-Fi 抖动) |
| `--skip-links` | 跳过符号链接/junction,防止死循环和报错 |
| `--partial-suffix .part` | 大文件断点保留,续传不从零开始 |
| `--filter-from filters.txt` | 排除依赖目录和系统垃圾(见 filters.txt) |
| `--log-file migrate.log` | 所有跳过/失败的文件都记录在案,事后可查 |
| 脚本外层 while 循环 | rclone 退出码非 0(有文件失败)→ 等 60 秒自动重跑,直到全部成功 |

**被其他程序锁定的文件怎么办?** 本轮跳过并记日志 → 下一轮自动重试。如果始终传不动(比如浏览器一直开着锁住缓存),日志里能看到,最后收尾时关掉相关程序再跑一次脚本即可——这是唯一可能需要你出手的情况,且发生在最后而不是中途。

## 五、完成与校验

脚本打印 `✅ 全部完成` 后,可选做一次严格校验(按需):

```powershell
# 在发送端运行,逐文件比对校验和(耗时较长,可只对重要目录做)
rclone check D:\projects :sftp,host=新电脑IP,port=2022,user=lanmigrate,pass=<obscured>:/projects --filter-from filters.txt
```

或者抽查几个重要项目目录的文件数和总大小是否一致。

## 六、注意事项

1. **filters.txt 里的 `dist/` `build/` 是一刀切排除**——M0 阶段图省事。如果你有非项目目录恰好叫这两个名字且内容重要,先在 filters.txt 里删掉对应行(这正是未来工具"上下文感知排除"要解决的问题)
2. `.git` 目录默认**保留**(占体积但历史有价值)
3. 两台电脑最好都插电、关闭睡眠(Windows:设置→电源→睡眠改为"从不";Mac:系统设置→显示器可以熄,勾选"防止自动进入睡眠")
4. 千兆有线 > Wi-Fi 6 > 普通 Wi-Fi。如果两台电脑都有网口,直连一根网线速度最稳(500GB 有线约 1.5~2 小时,Wi-Fi 可能 4~10 小时)
